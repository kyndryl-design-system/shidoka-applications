Interactive **two- or three-column** layout with **vertical `kyn-divider`** (**`drag-handle`**) and draggable column widths. The helpers **`split-view-pattern`** and **`split-view-code-rail`** ship with this Storybook pattern only‚Äîthey are **not** published `kyn-*` components. To reuse in an application, **vendor the pattern** (copy the files below) or reimplement the layout with **`kyn-divider`** and flex.

## How to explore this pattern

1. **This page** ‚Äî copy-paste imports, vendor checklist, API, resize constants.
2. **Stories** under **Patterns / Split View** ‚Äî **Two Pane Implemented** is the shortest copy-paste template (**issue detail** in **`slot="pane-1"`**, same body as **Three Pane Implemented**‚Äôs center pane, plus **`split-view-code-rail`** + **`kyn-block-code-view`** in **`slot="pane-2"`**); **Two Pane** / **Three Pane** use controls to switch column count; **Three Pane Implemented** adds the issue list + third-pane code rail (heavy Canvas‚Äîcopy **Two Pane Implemented** for a compact two-pane sample).
3. **Canvas ‚Üí Code** ‚Äî runnable Lit source. Imports below match **`splitView.stories.js`** (same folder as this doc).

## Copy-paste: this repo (Storybook Vite)

Use these paths from **`src/stories/splitView/`** (or adjust relative to your file). The **`virtual:split-view-css`** module is provided by **`.storybook/main.js`** only‚ÄîStorybook resolves it at build time.

**1. Register once** (module top level or app bootstrap):

```javascript
import { html, unsafeCSS } from 'lit';
import { registerSplitViewPattern } from './splitViewPattern';
import { registerSplitViewCodeRail } from './splitViewCodeRail';
import {
  SPLIT_VIEW_PATTERN_SHADOW_CSS,
  SPLIT_VIEW_CODE_RAIL_SHADOW_CSS,
  SPLIT_VIEW_BLOCK_CODE_FLUSH_CSS,
  SPLIT_VIEW_STORY_LIGHT_DOM_CSS,
} from 'virtual:split-view-css';

const splitViewStyles = html`<style>
  ${unsafeCSS(SPLIT_VIEW_STORY_LIGHT_DOM_CSS)}
</style>`;

registerSplitViewPattern(SPLIT_VIEW_PATTERN_SHADOW_CSS);
registerSplitViewCodeRail(
  SPLIT_VIEW_CODE_RAIL_SHADOW_CSS,
  SPLIT_VIEW_BLOCK_CODE_FLUSH_CSS
);
```

**2. Import primitives** used by your panes (pattern already imports **`kyn-divider`** internally; you still need **`kyn-block-code-view`** if you use the code rail):

```javascript
import '../../components/reusable/divider/divider';
import '../../components/reusable/blockCodeView/blockCodeView';
```

**3. Render** (wrap in `.split-view-shell`; use `slot="pane-3"` only when `.panes=${3}`):

```javascript
html`
  ${splitViewStyles}
  <div class="split-view-shell">
    <split-view-pattern .panes=${3} @on-resize=${(e) => console.log(e.detail)}>
      <div slot="pane-1">‚Ä¶</div>
      <div slot="pane-2">‚Ä¶</div>
      <div slot="pane-3" class="pane-3">
        <split-view-code-rail>
          <kyn-block-code-view
            darkTheme="dark"
            language="javascript"
            codeSnippet=${'const x = 1;'}
          ></kyn-block-code-view>
        </split-view-code-rail>
      </div>
    </split-view-pattern>
  </div>
`;
```

## Vendor checklist (app / package outside Storybook)

Copy or mirror these paths from **`src/stories/splitView/`**:

| File                       | Role                                                                                    |
| -------------------------- | --------------------------------------------------------------------------------------- |
| **`splitViewPattern.ts`**  | Defines `<split-view-pattern>`; calls `registerSplitViewPattern(css)`.                  |
| **`splitViewCodeRail.ts`** | Defines `<split-view-code-rail>`; calls `registerSplitViewCodeRail(hostCss, flushCss)`. |
| **`splitView.scss`**       | Single SCSS source with `/* @split-view-section: ‚Ä¶ */` markers (see below).             |

**CSS strings:** The Storybook Vite plugin reads **`splitView.scss`**, splits on **`@split-view-section`**, compiles each chunk with Sass, and exports four constants. Section keys must exist: **`pattern`**, **`railHost`**, **`blockFlush`**, **`storyLight`**. Plugin implementation: **`.storybook/main.js`** (`vitePluginSplitViewUnifiedScss`).

**Outside Storybook you must:**

1. Supply the same four strings to **`registerSplitViewPattern`**, **`registerSplitViewCodeRail`**, and your light-DOM `<style>` (same mapping as the virtual module: pattern ‚Üí first registration; railHost + blockFlush ‚Üí code rail registration; storyLight ‚Üí light DOM wrapper styles).
2. Either **port the plugin** (compile each SCSS section the same way) or **manually** extract the four blocks from **`splitView.scss`** and compile them in your build (preserve **`loadPaths`** so `@use` / imports resolve like Storybook).

**Types:** `declare module 'virtual:split-view-css'` in **`declarations.d.ts`** documents the four exports for editors; your app can re-export the same shape after vendoring.

## API

|                                 |                                                                                                                                                                                                                                                                                                                                                       |
| ------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`.panes`**                    | `2` or `3`. **Default:** `3` if omitted on `<split-view-pattern>`.                                                                                                                                                                                                                                                                                    |
| **`.invertStartDividerGrip`**   | When `true`, adds **`right-inverted-handle`** on the divider after the start pane (light + dark **`pane-2`**, e.g. code). Attribute **`invert-start-divider-grip`**. **Default:** `false`.                                                                                                                                                            |
| **`.invertEndDividerGrip`**     | When `true`, adds **`right-inverted-handle`** on the divider before the end pane (**`panes === 3`**, light primary + dark **`pane-3`**). Attribute **`invert-end-divider-grip`**. **Default:** `false`.                                                                                                                                               |
| **`.defaultStartPaneWidthPx`**  | Initial **start** pane width in **px** (`pane-1`). Attribute **`default-start-pane-width`**. Takes precedence over **`defaultStartPaneFraction`**. **Clamped** to min pane + center rules.                                                                                                                                                            |
| **`.defaultStartPaneFraction`** | Initial start width as a **fraction** of the track (e.g. `0.39` for ~40% start). Attribute **`default-start-pane-fraction`**. Used when **`defaultStartPaneWidthPx`** is not set; otherwise defaults to ~36% (two-pane) or ~34% (three-pane).                                                                                                         |
| **`.defaultEndPaneWidthPx`**    | Initial **end** pane width in **px** (`pane-3`) when **`panes === 3`**. Attribute **`default-end-pane-width`**. If omitted, ~28% of the track; **clamped** like **`defaultStartPaneWidthPx`**. Ignored when **`panes === 2`**. The **primary** pane (`pane-2`) is always flex ‚Äî its width is whatever space remains between start, dividers, and end. |
| **`slot="pane-1"`**             | Start column (fixed width; draggable).                                                                                                                                                                                                                                                                                                                |
| **`slot="pane-2"`**             | Center column (flex; grows/shrinks when dragging).                                                                                                                                                                                                                                                                                                    |
| **`slot="pane-3"`**             | End column; render only when **`.panes=${3}`**.                                                                                                                                                                                                                                                                                                       |
| **`@on-resize`**                | Fires **after** a divider drag ends (**pointerup** / **pointercancel**). **`event.detail`:** `{ leftPx: number, rightPx?: number }` ‚Äî **`rightPx`** is set only when **`.panes === 3`**.                                                                                                                                                              |

**Shadow **`part`** names** (for `::part(...)` styling if you expose it from the host): **`split-view`**, **`pane-start`**, **`pane-end`**.

## Resize behavior (implementation constants)

Defined in **`splitViewPattern.ts`**: outer panes and center column enforce minimum pixel widths ‚Äî **120px** minimum per outer pane, **160px** minimum center, **8px** divider track. Dragging clamps within those constraints.

**Pointer:** Resize uses **pointer** events on the dividers; there is **no keyboard** resize in this pattern.

## Layout behavior

Panes use flex; **slotted content** should fill pane height so **inner** regions (lists, detail, code) scroll. Put long content in a child with **`overflow: auto`** rather than growing past the split-view height.

## Mobile & accessibility (constraints)

**Mobile / responsive**

- There is **no** breakpoint that stacks columns or swaps to a drawer; the pattern is always a **horizontal** flex row. For small screens, **vendor** a single-column layout, tabs, or a drawer, or hide secondary panes.
- **Minimum widths** in **`splitViewPattern.ts`** (outer panes **120px**, center **160px**, **8px** dividers) imply a **minimum track width** before clamping squeezes the center‚Äîtest on **narrow** viewports (~320px).
- **`.split-view`** uses **`min-height: min(640px, 82vh)`** by default; under **`max-width: 480px`** the pattern SCSS uses a **shorter** min-height (**`min(480px, 72vh)`**) so the block is less dominant on short phones.
- **`.split-view-shell`** (Storybook) uses **`max()`** with **`env(safe-area-inset-*)`** so padding clears notches; copy that approach when embedding full-bleed on devices.
- The draggable gutter is **8px** wide‚Äîbelow common **44px** touch-target guidance. For production, consider a **wider invisible hit area** (without changing **`DIVIDER_PX`** clamp math) or document **pointer-only** resize.
- **Issue list** (Three Pane Implemented): **`.status-widget`** left accents use the Figma severity mapping (informational / error / warning / success) and heavy badges; they **do not** match **`kyn-widget`** left-border tokens. See **`splitView.scss`**.

**Accessibility**

- Resize is **pointer-only** (see **Resize behavior**). There is **no** keyboard or screen-reader-driven resize in the helper‚Äîprovide **an equivalent** in the product if you must meet **WCAG** dragging / keyboard expectations (e.g. width inputs, presets).
- **`kyn-divider`** with **`drag-handle`** exposes **`role="separator"`**, **`aria-orientation`**, and a default **`aria-label`** (‚ÄúResize panels‚Äù). Override with **`resizeLabel`** on **`kyn-divider`** where copy should be specific.
- **Hairline / grip** motion respects **`prefers-reduced-motion`** in **`divider.scss`** and split-view light-DOM hairline styles.
