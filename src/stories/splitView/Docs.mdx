Interactive **two- or three-column** layout with **vertical `kyn-divider`** (**`drag-handle`**), keyboard-resizable separators, and a compact **single-active-pane + tabs** mode on narrow widths. The helpers **`split-view-pattern`** and **`split-view-code-rail`** ship with this Storybook pattern only—they are **not** published `kyn-*` components. To reuse in an application, **vendor the pattern** (copy the files below) or reimplement the layout with **`kyn-divider`** and flex.

## How to explore this pattern

1. **This page** — copy-paste imports, vendor checklist, API, resize constants.
2. **Stories** under **Patterns / Split View** — **Two Pane** and **Three Pane** are the starter canvases; **Two Pane Implemented** is the shortest copy-paste template (**issue detail** in **`slot="pane-1"`** plus **`split-view-code-rail`** + **`kyn-block-code-view`** in **`slot="pane-2"`**); **Three Pane Implemented** adds the issue list + third-pane code rail; **Compact Three Pane** demonstrates the narrow-screen tabs model.
3. **Canvas → Code** — runnable Lit source. Imports below match **`splitView.stories.js`** (same folder as this doc).

## Copy-paste: this repo (Storybook Vite)

Use these paths from **`src/stories/splitView/`** (or adjust relative to your file). The split view styles now come from four direct `?inline` SCSS imports in **`registerSplitView.ts`**, so there is no Storybook-only virtual CSS module to preserve.

**1. Register once** (module top level or app bootstrap) with the convenience helper:

```javascript
import { splitViewStoryStyles } from './registerSplitView';
```

**2. Import primitives** used by your panes (pattern already imports **`kyn-divider`** internally; you still need **`kyn-block-code-view`** if you use the code rail):

```javascript
import '../../components/reusable/divider/divider';
import '../../components/reusable/blockCodeView/blockCodeView';
```

**3. Render** (wrap in `.split-view-shell`; use `slot="pane-3"` only when `.panes=${3}`):

```javascript
html`
  ${splitViewStoryStyles}
  <div class="split-view-shell">
    <split-view-pattern
      .panes=${3}
      pane-1-label="Issue list"
      pane-2-label="Issue detail"
      pane-3-label="Code rail"
      @split-view-resize=${(e) => console.log(e.detail)}
    >
      <div slot="pane-1">…</div>
      <div slot="pane-2">…</div>
      <div slot="pane-3" class="pane-3">
        <split-view-code-rail>
          <kyn-block-code-view
            darkTheme="dark"
            language="javascript"
            codeSnippet=${'const x = 1;'}
          ></kyn-block-code-view>
        </split-view-code-rail>
      </div>
    </split-view-pattern>
  </div>
`;
```

## Vendor checklist (app / package outside Storybook)

Copy or mirror these paths from **`src/stories/splitView/`**:

| File                           | Role                                                                                          |
| ------------------------------ | --------------------------------------------------------------------------------------------- |
| **`registerSplitView.ts`**     | Storybook-only convenience wrapper: registers helpers and exports **`splitViewStoryStyles`**. |
| **`splitViewPattern.ts`**      | Defines `<split-view-pattern>`; calls `registerSplitViewPattern(css)`.                        |
| **`splitViewCodeRail.ts`**     | Defines `<split-view-code-rail>`; calls `registerSplitViewCodeRail(hostCss, flushCss)`.       |
| **`splitViewPattern.scss`**    | Shadow styles for `<split-view-pattern>`.                                                     |
| **`splitViewCodeRail.scss`**   | Shadow styles for `<split-view-code-rail>`.                                                   |
| **`splitViewBlockFlush.scss`** | Adopted styles for the slotted `kyn-block-code-view` flush rail treatment.                    |
| **`splitViewStory.scss`**      | Story/light-DOM layout and demo styling used by `splitViewStoryStyles`.                       |

**CSS strings:** `registerSplitView.ts` imports each SCSS file with **`?inline`** so Vite compiles them to strings directly. The mapping is one-to-one: pattern → `registerSplitViewPattern`, code rail → `registerSplitViewCodeRail`, block flush → `registerSplitViewCodeRail`, story styles → `splitViewStoryStyles`.

**Outside Storybook you must:**

1. Supply the same four compiled strings to **`registerSplitViewPattern`**, **`registerSplitViewCodeRail`**, and your light-DOM `<style>` (pattern → splitter shadow root; code rail + block flush → rail registration; story styles → light DOM wrapper styles).
2. Either keep the four SCSS files and let your build compile `?inline` imports, or compile those files to strings by whatever mechanism your app already uses for Lit CSS.

## API

|                                                   |                                                                                                                                                                                                                                                                                                                                                       |
| ------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`.panes`**                                      | `2` or `3`. **Default:** `2` if omitted on `<split-view-pattern>`.                                                                                                                                                                                                                                                                                    |
| **`.invertStartDividerGrip`**                     | When `true`, adds **`right-inverted-handle`** on the divider after the start pane (light + dark **`pane-2`**, e.g. code). Attribute **`invert-start-divider-grip`**. **Default:** `false`.                                                                                                                                                            |
| **`.invertEndDividerGrip`**                       | When `true`, adds **`right-inverted-handle`** on the divider before the end pane (**`panes === 3`**, light primary + dark **`pane-3`**). Attribute **`invert-end-divider-grip`**. **Default:** `false`.                                                                                                                                               |
| **`.defaultStartPaneWidthPx`**                    | Initial **start** pane width in **px** (`pane-1`). Attribute **`default-start-pane-width`**. Takes precedence over **`defaultStartPaneFraction`**. **Clamped** to min pane + center rules.                                                                                                                                                            |
| **`.defaultStartPaneFraction`**                   | Initial start width as a **fraction** of the track (e.g. `0.39` for ~40% start). Attribute **`default-start-pane-fraction`**. Used when **`defaultStartPaneWidthPx`** is not set; otherwise defaults to ~36% (two-pane) or ~34% (three-pane).                                                                                                         |
| **`.defaultEndPaneWidthPx`**                      | Initial **end** pane width in **px** (`pane-3`) when **`panes === 3`**. Attribute **`default-end-pane-width`**. If omitted, ~28% of the track; **clamped** like **`defaultStartPaneWidthPx`**. Ignored when **`panes === 2`**. The **primary** pane (`pane-2`) is always flex — its width is whatever space remains between start, dividers, and end. |
| **`.pane1Label` / `.pane2Label` / `.pane3Label`** | Labels for the compact-mode tab buttons and supporting accessible text. Attributes **`pane-1-label`**, **`pane-2-label`**, **`pane-3-label`**.                                                                                                                                                                                                        |
| **`.compactBreakpointPx`**                        | Switches from desktop split columns to compact tabs mode when the host width is at or below this px value. Attribute **`compact-breakpoint`**. **Default:** `768`.                                                                                                                                                                                    |
| **`.compactActivePane`**                          | Visible pane while the component is in compact mode. Attribute **`compact-active-pane`**. Values normalize automatically to a visible pane for the current **`.panes`** count.                                                                                                                                                                        |
| **`.startResizeLabel` / `.endResizeLabel`**       | Accessible names for the keyboard-focusable separators. Attributes **`start-resize-label`** and **`end-resize-label`**.                                                                                                                                                                                                                               |
| **`slot="pane-1"`**                               | Start column (fixed width; draggable).                                                                                                                                                                                                                                                                                                                |
| **`slot="pane-2"`**                               | Center column (flex; grows/shrinks when dragging).                                                                                                                                                                                                                                                                                                    |
| **`slot="pane-3"`**                               | End column; render only when **`.panes=${3}`**.                                                                                                                                                                                                                                                                                                       |
| **`@split-view-resize`**                          | Primary resize event. Fires after pointer drag ends and after keyboard resize steps. **`event.detail`:** `{ leftPx: number, rightPx?: number, source: 'pointer' \| 'keyboard' }` — **`rightPx`** is set only when **`.panes === 3`**.                                                                                                                 |
| **`@on-resize`**                                  | Legacy alias of **`split-view-resize`** kept for backwards compatibility with existing stories and consumers.                                                                                                                                                                                                                                         |
| **`@split-view-pane-change`**                     | Fires when compact-mode pane switching changes the active pane. **`event.detail`:** `{ activePane: 1 \| 2 \| 3, compact: boolean }`.                                                                                                                                                                                                                  |

**Shadow **`part`** names** (for `::part(...)` styling if you expose it from the host): **`split-view`**, **`pane-start`**, **`pane-end`**.

## Resize behavior (implementation constants)

Defined in **`splitViewPattern.ts`**: outer panes and center column enforce minimum pixel widths — **120px** minimum per outer pane, **160px** minimum center, **8px** divider track. Dragging clamps within those constraints.

**Pointer:** Resize uses **pointer** events on the separators.

**Keyboard:** Focus a separator and use **Left / Right Arrow** to resize. Hold **Shift** for larger steps. Use **Home / End** to jump to the minimum / maximum allowed width for that divider.

## Layout behavior

Panes use flex; **slotted content** should fill pane height so **inner** regions (lists, detail, code) scroll. Put long content in a child with **`overflow: auto`** rather than growing past the split-view height.

## Mobile & accessibility (constraints)

**Mobile / responsive**

- At or below **`.compactBreakpointPx`** (default **768px**), the layout switches into a **single active pane** model with **tabs** above the content area. This avoids forcing 2-3 horizontal columns into narrow widths.
- **Minimum widths** in **`splitViewPattern.ts`** (outer panes **120px**, center **160px**, **8px** dividers) imply a **minimum track width** before clamping squeezes the center—test on **narrow** viewports (~320px).
- **`.split-view`** uses **`min-height: min(640px, 82vh)`** by default; under **`max-width: 480px`** the pattern SCSS uses a **shorter** min-height (**`min(480px, 72vh)`**) so the block is less dominant on short phones.
- **`.split-view-shell`** (Storybook) uses **`max()`** with **`env(safe-area-inset-*)`** so padding clears notches; copy that approach when embedding full-bleed on devices.
- The draggable gutter is still **8px** wide in desktop/tablet mode. Compact mode reduces the need for dragging on small touch targets, but test your product context if touch resizing remains important on medium viewports.
- **Issue list** (Three Pane Implemented): list rows now use **`kyn-widget`** for the left accent treatment and keep a custom inner row layout so the icon/title/timestamp match the Figma composition.

**Accessibility**

- Separators are keyboard-focusable and expose **`role="separator"`**, **`aria-orientation="vertical"`**, and current sizing values from **`split-view-pattern`**. Use **`startResizeLabel`** / **`endResizeLabel`** when product copy should be more specific.
- Compact mode renders a semantic **tablist** with keyboard navigation (**Left / Right / Home / End**) and associated **tab panels**.
- **`kyn-divider`** stays the visual drag affordance. In Split View it is rendered decoratively while the parent separator owns the interaction and semantics.
- **Hairline / grip** motion respects **`prefers-reduced-motion`** in **`divider.scss`** and split-view light-DOM hairline styles.
