# Icons

<br />

## Carbon Icons

Most of our icon usage will come from IBM Carbon.

### Framework-specific Icons

It is recommended to use framework-specific versions of the Carbon components.

- [React](https://github.com/carbon-design-system/carbon/tree/main/packages/icons-react)
- [Svelte](https://github.com/carbon-design-system/carbon-icons-svelte)
- [Vue](https://github.com/carbon-design-system/carbon/tree/main/packages/icons-vue)
- [Angular](https://github.com/carbon-design-system/carbon-icons-angular)

### Vanilla Icons

Use the `<kyn-icon>` component to make using [vanilla Carbon Icons](https://github.com/carbon-design-system/carbon/tree/main/packages/icons) (bundled with this library) easy. This is more intended for vanilla/Lit usage internally, but should work anywhere.

```js
import { Icon } from '@kyndryl-design-system/web-components';
import overflowIcon from '@carbon/icons/es/overflow-menu--vertical/24';
```

```html
<kyn-icon .icon="${overflowIcon}"></kyn-icon>
```

## Kyndryl Icons

Custom icons are SVG files that you can import directly from this library.

### Example Usage

It is recommended to use an SVG loader to inject the SVG as inline HTML into your app.

- Example Loaders:
  - [Rollup](https://www.npmjs.com/package/rollup-plugin-svg)
  - [Webpack](https://www.npmjs.com/package/svg-inline-loader)

SVG files can then be imported like this:

```js
import iconName from '@kyndryl-design-system/web-components/assets/svg/<filename>.svg';
```

The method for placing them in your app at this point will vary based on the framework/loader.
